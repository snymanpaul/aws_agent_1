---
name: Commit after every level
description: Always commit after each completed level for traceability
type: feedback
---

Create a git commit after completing each level (L28, L29, etc.) before starting the next.

**Why:** Proper traceability — each level's work should be an isolated, auditable unit in git history.

**How to apply:** After the level file runs cleanly and the learning plan is updated, commit with a message like "feat(L28): SDK Advances — Concurrency, MCP Tasks, Hooks" before touching L29 work.
